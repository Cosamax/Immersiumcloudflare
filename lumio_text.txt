Base de connaissances :
Base de connaissances — Internet et le Web
Notion 1 — Internet
Définition : Internet est un réseau mondial qui relie des milliards d'ordinateurs, de téléphones et de serveurs entre eux. C'est une infrastructure physique faite de câbles, de routeurs et de serveurs qui permet à tous ces appareils de communiquer. Internet existe depuis les années 1960 et le grand public y a accès depuis les années 1990.
Analogie : Imagine le réseau routier d'un pays. Il y a des autoroutes, des routes départementales, des petites rues. Tous ces chemins sont connectés entre eux et permettent à des millions de véhicules de circuler. Internet, c'est exactement la même chose — mais pour les données numériques.
Exemple LUMIO : Quand une cliente de Sophie envoie un email pour commander une bougie, cet email voyage sur Internet — exactement comme un colis voyage sur les routes. Sans Internet, cet email n'arriverait jamais.
Notion 2 — Le Web
Définition : Le Web (World Wide Web) est un service qui fonctionne sur Internet. C'est l'ensemble des pages et des sites accessibles via un navigateur (Chrome, Firefox, Safari...). Le Web a été inventé en 1989 par Tim Berners-Lee. Attention : le Web et Internet ne sont pas la même chose. Internet est l'autoroute. Le Web est un type de véhicule qui circule sur cette autoroute.
Analogie : Sur cette autoroute (Internet) circulent des voitures (le Web), des camions (les emails), des bus (le streaming), des motos (les appels en ligne). Tous utilisent la même autoroute mais ce sont des véhicules différents avec des usages différents.
Exemple LUMIO : Quand Sophie consulte Instagram sur son téléphone, elle utilise Internet. Quand elle ouvre le site d'un fournisseur dans son navigateur, elle utilise le Web. Quand elle envoie un email à une cliente, elle utilise Internet mais pas le Web.
Notion 3 — Le navigateur et l'URL
Définition : Un navigateur est le logiciel qui te permet de consulter des sites web. Les plus connus sont Google Chrome, Mozilla Firefox, Safari et Microsoft Edge. Une URL (Uniform Resource Locator) est l'adresse d'une page web. C'est ce que tu tapes dans la barre en haut du navigateur pour aller sur un site.
Analogie : Le navigateur, c'est ta voiture. L'URL, c'est l'adresse GPS que tu tapes pour arriver à destination. Sans adresse GPS, tu ne sais pas où aller. Sans navigateur, tu ne peux pas circuler sur le Web.
Exemple LUMIO : Quand le futur site de Sophie sera en ligne, ses clientes taperont « www.lumio-bougies.fr » dans leur navigateur pour y accéder. Cette URL, c'est l'adresse digitale de l'atelier.
Notion 4 — Le nom de domaine et l'hébergement
Définition : Le nom de domaine est l'adresse unique d'un site web sur Internet. Il se compose d'un nom choisi par l'entreprise et d'une extension (.fr, .com, .org...). Il se loue chaque année pour environ 12€. L'hébergement web est le service qui stocke tous les fichiers du site sur un serveur connecté en permanence à Internet.
Analogie : Le nom de domaine, c'est l'adresse postale de ta maison. L'hébergement, c'est la maison elle-même. Tu peux avoir la plus belle adresse du monde — si la maison n'existe pas, personne ne peut entrer.
Exemple LUMIO : Sophie doit acheter le nom de domaine « lumio-bougies.fr » (12€/an) ET louer un hébergement (environ 8€/mois) pour que son site soit visible sur Internet.
Notion 5 — Le HTTPS
Définition : HTTPS signifie HyperText Transfer Protocol Secure. C'est la version sécurisée du protocole qui permet d'échanger des données sur le Web. Quand un site est en HTTPS, les données échangées entre le visiteur et le site sont chiffrées. Tu reconnais un site HTTPS au petit cadenas affiché dans la barre de ton navigateur.
Analogie : HTTP, c'est comme envoyer une lettre dans une enveloppe transparente. N'importe qui peut lire ce qu'il y a dedans. HTTPS, c'est une enveloppe opaque, scellée et cryptée. Seul le destinataire peut l'ouvrir.
Exemple LUMIO : Si le site de Sophie n'est pas en HTTPS, Google affiche une alerte « site non sécurisé » à ses visiteuses. La plupart partiront immédiatement. En plus, Google pénalise les sites non sécurisés dans ses résultats.
Vidéo recommandée : Comment fonctionne Internet ?
Ta mission : Lis attentivement les notions ci-dessus. Sophie va tester ta compréhension à travers trois exercices progressifs.
Options :
✓ Option 1 : Internet est le réseau physique mondial qui relie tous les appareils entre eux. Le Web est un service qui fonctionne sur ce réseau et permet d'accéder à des pages via un navigateur.
Feedback : Exactement. Internet c'est l'infrastructure — les câbles, les serveurs, les routeurs. Le Web c'est un service parmi d'autres qui utilise cette infrastructure. L'email, le streaming, les appels vidéo utilisent aussi Internet — mais ce ne sont pas du Web. Pour LUMIO, ça veut dire que créer un site web, c'est créer quelque chose sur le Web — qui lui-même repose sur Internet.
✗ Option 2 : Internet et le Web sont deux noms différents pour désigner la même chose — le réseau mondial de communication.
Feedback : Non. C'est justement l'erreur que Sophie voulait éviter. Internet c'est l'autoroute — le réseau physique mondial. Le Web c'est un type de véhicule qui circule sur cette autoroute. D'autres véhicules utilisent la même autoroute : les emails, les appels vidéo, le streaming.
✗ Option 3 : Le Web est le réseau physique mondial. Internet est le service qui permet de naviguer sur des pages via un navigateur.
Feedback : Non, tu as inversé les deux. Internet c'est le réseau physique — les câbles sous-marins, les serveurs, les routeurs. Le Web, inventé par Tim Berners-Lee en 1989, c'est le service qui s'appuie sur cette infrastructure pour afficher des pages dans un navigateur.
✗ Option 4 : Internet est un service payant accessible via abonnement. Le Web est la version gratuite accessible à tous.
Feedback : Non. La distinction entre Internet et le Web n'a rien à voir avec le coût ou l'accessibilité. L'abonnement internet que tu paies chez un opérateur te donne accès au réseau — mais ça ne définit pas ce qu'est Internet ni ce qu'est le Web.
À retenir :

Base de connaissances :
Base de connaissances — Stratégie digitale, Objectif SMART et KPI
Notion 1 — La stratégie digitale
Définition : Une stratégie digitale est un plan d'action structuré qui définit comment une entreprise va utiliser les outils digitaux pour atteindre ses objectifs commerciaux. Elle répond à quatre questions : Pourquoi (quels objectifs ?), Pour qui (quelles cibles ?), Quoi (quels messages ?), Comment (quels canaux et outils ?).
Analogie : Un GPS : tu entres ta destination avant de partir. Il calcule le meilleur itinéraire selon ta position, le trafic et tes contraintes. Une stratégie digitale fait pareil : tu définis où tu veux aller, d'où tu pars et comment tu vas y arriver — avant d'agir.
Exemple LUMIO : Sans stratégie : Sophie publie des photos « quand elle a le temps », sans objectif ni planning. Avec stratégie : elle publie 3 fois par semaine selon un planning précis, avec pour objectif 500 abonnés en 3 mois et 10 commandes via Instagram par mois.
Notion 2 — L'objectif SMART
Définition : Un objectif SMART respecte cinq critères : Spécifique (précis et sans ambiguïté), Mesurable (quantifiable avec des chiffres), Atteignable (ambitieux mais réaliste), Réaliste (adapté aux ressources disponibles), Temporel (avec une date limite). Sans ces cinq critères, un objectif est un vœu — pas un outil de pilotage.
Analogie : « Je veux être en meilleure forme » n'est pas SMART. « Courir 5km sans m'arrêter d'ici le 1er juin » l'est. La différence : un chiffre précis et une date.
Exemple LUMIO : Pas SMART : « Je veux que LUMIO soit connue sur Instagram. » SMART : « Atteindre 500 abonnés Instagram et 15 commandes via le lien en bio d'ici le 31 mars prochain. »
Notion 3 — Le KPI
Définition : Un KPI (Key Performance Indicator) ou Indicateur Clé de Performance est un chiffre qui permet de mesurer si on atteint ses objectifs. Chaque objectif a ses KPI. Sans KPI, impossible de savoir si la stratégie fonctionne. Les KPI doivent être simples, accessibles facilement et vérifiés à fréquence régulière.

Notion 1 — Les Core Web Vitals
Définition : Les Core Web Vitals sont trois indicateurs de performance technique définis par Google. LCP (Largest Contentful Paint) : temps de chargement du plus grand élément visible — seuil sous 2,5 secondes. FID (First Input Delay) : temps de réponse au premier clic — seuil sous 100ms. Notion 2 — Le maillage interne avancé
Définition : Le maillage interne avancé est la stratégie qui consiste à structurer les liens entre les pages d'un site pour transférer l'autorité SEO vers les pages les plus importantes. Les pages avec le plus de liens entrants ont plus d'autorité aux yeux de Google.
Analogie : Dans une ville, les rues principales qui reçoivent le plus de trafic et vers lesquelles convergent le plus de routes secondaires sont les plus importantes. Google comprend l'importance d'une page de la même façon.
Exemple LUMIO : La page Boutique de LUMIO doit recevoir des liens depuis toutes les autres pages importantes : accueil, articles de blog, page À propos.
Notion 3 — L'autorité de domaine et les backlinks
Définition : L'autorité de domaine est un score (0 à 100) qui mesure la crédibilité d'un site. Elle se construit principalement par les backlinks — les liens provenant d'autres sites. Un lien depuis un site reconnu vaut bien plus que des dizaines de liens depuis des sites peu crédibles.
Analogie : La recommandation d'un expert reconnu vaut infiniment plus que dix recommandations de parfaits inconnus. Si le guide Michelin recommande un restaurant, c'est plus puissant que 50 avis d'inconnus.
Exemple LUMIO : Un article dans Le Progrès qui mentionne LUMIO et fait un lien vers le site apporte plus d'autorité que 50 annuaires génériques. Stratégie : envoyer des bougies à des blogueurs déco lyonnais en échange d'un article.

Base de connaissances — Fidélisation digitale, Séquence email automatisée et LTV
Notion 1 — La fidélisation digitale
Définition : La fidélisation digitale consiste à mettre en place des actions pour qu'un client qui a déjà acheté revienne, achète à nouveau et recommande la marque. Les leviers : email marketing post-achat, offres réservées, programmes de parrainage, contenus exclusifs et remarketing. Un client fidèle a un CAC de 0€ sur ses achats répétés.
Analogie : Un restaurant qui envoie une carte d'anniversaire avec une offre spéciale à ses clients réguliers, qui se souvient de leurs préférences et qui les appelle par leur prénom fidélise sans effort apparent.
Exemple LUMIO : Si Sophie met en place un email de suivi post-achat, un email 30 jours après avec une recommandation et un programme de parrainage, ses 30 clientes deviennent 60 en quelques mois.
Notion 2 — La séquence email automatisée
Définition : Une séquence email automatisée (ou workflow) est une série d'emails envoyés automatiquement à des moments précis selon des déclencheurs (trigger). Mailchimp permet de créer ces séquences gratuitement jusqu'à 500 contacts.
Analogie : Un bon vendeur de boutique physique suit naturellement ses clients : il les accueille, les rappelle une semaine après pour s'assurer de leur satisfaction, et les prévient des nouveautés. La séquence automatisée fait exactement ça — de façon automatique et à grande échelle.
Exemple LUMIO : Séquence post-achat LUMIO : Email 1 (immédiat) : « Merci pour votre commande ! Sophie fabrique votre bougie aujourd'hui. » Email 2 (J+7) : « Comment entretenir votre bougie pour qu'elle dure plus longtemps. » Email 3 (J+30) : « Découvrez la nouvelle collection + 10% pour nos fidèles. »
Notion 3 — La valeur vie client (LTV)
Définition : La LTV (Lifetime Value) est le montant total qu'un client dépense chez une marque sur toute la durée de sa relation. LTV = valeur moyenne d'achat × fréquence d'achat × durée de la relation. La règle : LTV doit être au moins 3 fois supérieure au CAC.
Analogie : Un abonné Netflix paie 15€/mois. S'il reste 3 ans, sa LTV est de 540€. Netflix peut donc investir jusqu'à 180€ pour acquérir un nouvel abonné et rester rentable.
Exemple LUMIO : LUMIO : 32€ × 3 fois/an × 2 ans = 192€. CAC actuel : 7,73€. Ratio LTV/CAC = 24,8 — excellent.

Base de connaissances — RGPD, Politique de confidentialité et Cookies
Notion 1 — Les obligations RGPD d'une TPE
Définition : Le RGPD impose à toute entreprise qui collecte des données personnelles : informer les personnes (politique de confidentialité), obtenir leur consentement, permettre l'exercice de leurs droits (accès, rectification, suppression), sécuriser les données, et notifier la CNIL en cas de violation dans les 72 heures.
Analogie : Le RGPD, c'est comme les règles d'hygiène dans un restaurant. Peu importe la taille du restaurant, les règles s'appliquent. LUMIO doit respecter le RGPD exactement comme une grande entreprise.
Exemple LUMIO : LUMIO collecte : emails (Mailchimp), comportement site (GA4), données navigation (pixel Meta). Ce qui manque probablement : politique de confidentialité, bandeau cookie conforme, mention légale complète.
Notion 2 — La politique de confidentialité
Définition : La politique de confidentialité est un document légal obligatoire qui explique aux visiteurs quelles données sont collectées, pourquoi, par qui, pendant combien de temps, et comment exercer leurs droits. Elle doit être accessible depuis toutes les pages du site. L'absence expose à une amende pouvant atteindre 4% du CA.
Analogie : La politique de confidentialité, c'est le contrat de confiance entre LUMIO et ses clientes. Quand une cliente donne son email, elle a le droit de savoir ce que LUMIO va en faire.
Exemple LUMIO : Politique minimale pour LUMIO : données collectées (email, prénom, comportement site), finalité (newsletter, suivi commandes), durée (3 ans après dernier achat), sous-traitants (Mailchimp, Google Analytics, Meta Pixel), droits (accès, rectification, suppression via contact@lumio.fr).
Notion 3 — Les cookies et le bandeau de consentement
Définition : Les cookies sont de petits fichiers déposés pour tracker le comportement. Depuis 2021 et les guidelines CNIL, un bandeau de consentement est obligatoire. Le bouton « refuser » doit être aussi visible que le bouton « accepter ». Un bandeau avec uniquement « Accepter » n'est pas conforme.
Analogie : Un bandeau cookie conforme, c'est comme un serveur qui demande avant de vous servir : « Voulez-vous de l'eau gazeuse ou plate ? » — avec un vrai choix.
Exemple LUMIO : Bandeau conforme pour LUMIO : « LUMIO utilise des cookies pour améliorer votre expérience. [Tout accepter] [Tout refuser] [Personnaliser] » — les trois boutons visuellement équivalents.

Notion 1 — Le plan stratégique annuel
Définition : Un plan stratégique digital annuel définit les objectifs, les moyens, le budget et le calendrier de toutes les actions marketing digitales sur 12 mois. Il se distingue du plan opérationnel mensuel par sa vision long terme et sa logique de priorisation.
Analogie : Un architecte ne commence pas à construire une maison sans plan. Il dessine d'abord l'ensemble du projet — fondations, structure, finitions — avant de commencer.
Exemple LUMIO : Le plan annuel de LUMIO organise l'année en 4 phases : T1 (Jan-Mar) Fondations et croissance organique, T2 (Avr-Jun) Fête des mères et collection printemps, T3 (Jul-Sep) Préparation de Noël, T4 (Oct-Déc) Saison haute — campagnes maximales.
Notion 2 — L'allocation budgétaire
Définition : L'allocation budgétaire est la répartition du budget marketing entre les différents leviers selon leur ROI attendu et leur rôle stratégique. Règle de base : 40% sur les leviers qui fonctionnent déjà, 40% sur les leviers à développer, 20% sur les tests et l'innovation.
Analogie : Un investisseur ne met pas tous ses œufs dans le même panier. Il diversifie : actifs sûrs (leviers éprouvés), actifs de croissance (leviers à développer), actifs spéculatifs (tests).
Exemple LUMIO : Budget annuel LUMIO 1 080€ : Google Ads 480€ (44%, ROAS éprouvé à 4,6), Meta Ads 240€ (22%, en développement), Création de contenu 240€ (22%, investissement SEO), Tests et innovation 120€ (12%).
Notion 3 — La gestion des risques
Définition : La gestion des risques consiste à identifier les facteurs qui peuvent perturber l'exécution du plan et à prévoir des réponses alternatives. Risques typiques : changement d'algorithme, augmentation des CPCs, indisponibilité du site, crise de réputation, concurrent agressif.
Analogie : Un capitaine de navire ne part pas en mer sans plan de tempête. Il connaît les risques probables selon la saison, a des procédures d'urgence pour chaque scénario.
Exemple LUMIO : Risque : Meta modifie son algorithme et le ROAS chute de 4,2 à 1,5. Plan B : réallouer le budget Meta vers Google Ads (ROAS plus stable) et augmenter la fréquence des emails à la liste existante (coût quasi nul).
